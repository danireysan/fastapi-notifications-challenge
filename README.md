# Decisions Taken for:

## Setup, Architecture and Design
- The project is made with a feature folder based clean architecture monolith to have a foundation for making a project that is easy to test and easy to refactor to microservices if needed
- Used UV as virtual environment because of its speed and versability
- We use the result pattern no have explicit contracts, the result pattern keeps the execution linear, making the code much easier to trace and debug
